Create a new file at lib/supabase/service.ts that exports a single function createServiceClient().

The file MUST:
- Import createClient from "@supabase/supabase-js" (NOT from @supabase/ssr)
- Export the function as createServiceClient (not createClient — avoids naming collision with server.ts and client.ts)
- Read NEXT_PUBLIC_SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY from process.env
- Throw a clear error if either env var is missing (do not silently fall back to anon key — the billing webhook's fallback `?? NEXT_PUBLIC_SUPABASE_ANON_KEY` is a weakness; do not replicate it)
- Pass auth: { persistSession: false, autoRefreshToken: false } to disable session handling entirely

Exact implementation:

```typescript
import { createClient } from "@supabase/supabase-js"

/**
 * Server-only Supabase admin client.
 *
 * Uses SUPABASE_SERVICE_ROLE_KEY to bypass RLS. NEVER use in client
 * components or browser-accessible routes. Safe for crons, webhooks,
 * and background jobs where there is no user session.
 */
export function createServiceClient() {
  const url = process.env.NEXT_PUBLIC_SUPABASE_URL
  const key = process.env.SUPABASE_SERVICE_ROLE_KEY
  if (!url || !key) {
    throw new Error(
      "Missing Supabase service credentials: NEXT_PUBLIC_SUPABASE_URL and SUPABASE_SERVICE_ROLE_KEY must both be set"
    )
  }
  return createClient(url, key, {
    auth: { persistSession: false, autoRefreshToken: false },
  })
}
```

No default export. No other exports. File is intentionally minimal.
